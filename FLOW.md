## 1. 프로젝트 개요

**TNFD(자연 관련 재무정보 공개 태스크포스)** 기준에 따라 기업의 지속가능성 보고서를 자동으로 진단하는 웹 애플리케이션입니다.

- **프론트엔드**: 단일 HTML 파일 (`index.html`) — 빌드 도구 없음, 순수 HTML/CSS/JS
- **백엔드**: Supabase (인증 + PostgreSQL DB + 파일 스토리지)
- **AI 엔진**: Anthropic Claude API (`claude-sonnet-4-5`)
- **배포**: Vercel (정적 파일 호스팅)

## 4. 인증 시스템

### 방식

Supabase Auth를 사용하되, **이메일 없이 아이디/비밀번호만** 사용합니다.

- 아이디는 내부적으로 `아이디@local.tnfd` 형식의 가짜 이메일로 변환하여 저장
- 이메일 검증 비활성화 (Supabase 대시보드에서 설정)

### 아이디 규칙

- 3~30자
- 영문 소문자, 숫자, `.`, `_`, `-` 만 허용
- 정규식: `/^[a-z0-9._-]{3,30}$/`

### 비밀번호 규칙

- 최소 6자

### 세션 관리

- Supabase SDK가 토큰 자동 갱신
- 재방문 시 자동 로그인 (`checkSession()`)
- 로그아웃 시 세션 삭제 후 페이지 새로고침

## 5. PDF 업로드 및 처리

### 업로드 입력 항목

| 항목 | 필수 | 설명 |
|------|------|------|
| 진단 대상 PDF | ✅ | 분석할 보고서 (최대 50MB) |
| BM 비교 PDF | ❌ | 벤치마크 회사 보고서 |
| 진단 대상 회사명 | ✅ | 텍스트 입력 |
| BM 회사명 | BM 업로드 시 | 텍스트 입력 |
| 산업 | ✅ | 16개 한국 산업 드롭다운 |
| 보고서 연도 | ✅ | 기본값: 현재 연도 |
| Anthropic API 키 | ✅ | `sk-ant-` 형식, DB에 저장 안 됨 |

### PDF 텍스트 추출

1. `PDF.js` 라이브러리로 전체 페이지 텍스트 추출
2. 페이지별로 `pdfTexts[]` 배열에 저장 (환각 검증에 활용)
3. 추출 텍스트가 500자 미만이면 오류 처리 (스캔 PDF 대응 불가 안내)

### 진단 시작 조건

- PDF 파일 업로드됨
- API 키 입력됨
→ 두 조건 모두 충족 시 "진단 시작" 버튼 활성화

## 6. Claude API 진단 실행

### 배치 처리 전략

125개 가이던스를 6개 배치로 나눠 API를 순차 호출합니다 (토큰 한도 관리).

| 배치 | 영역 | 가이던스 수 |
|------|------|-----------|
| 1 | G-A + G-B (이사회/경영진 거버넌스) | ~33개 |
| 2 | G-C (이해관계자 참여) | ~24개 |
| 3 | S-A + S-B (이슈 식별 + 사업 영향) | ~23개 |
| 4 | S-C + S-D + R (시나리오 + 위치 + 평가) | ~23개 |
| 5 | M-A + M-B (지표) | ~20개 |
| 6 | M-C (목표 설정) | ~17개 |

BM PDF가 있으면 동일 배치를 병렬 실행합니다.

### 진행률 표시

```
0% → PDF 업로드 중
10% → 배치 1 시작
30% → 배치 2
50% → 배치 3
65% → 배치 4
80% → 배치 5
90% → 배치 6
100% → 저장 완료
```

### 시스템 프롬프트 (평가 기준)

**이행 (Met)** — 아래 모두 충족:
- 보고서에 명시적·구체적으로 기술됨
- 정책/프로세스/시스템 존재 확인 가능
- 추상적 선언이 아닌 실제 내용/결과 가시
- 점수: 70~95

**부분이행 (Partial)** — 하나 이상 해당:
- 주제는 다루나 요건의 약 50%만 충족
- 계획/정책은 있으나 실행 세부사항 없음
- 부분적·일반적 수준의 정보
- 점수: 30~65

**미이행 (Not)** — 하나라도 해당:
- 핵심 요건이 직접 언급되지 않음
- 키워드만 등장하고 가이던스와 무관
- 다른 주제(예: 자연 대신 기후만 다룸)
- 추론/해석이 필요한 경우 → 미이행 처리
- 점수: 0~20

### 발췌문(Excerpt) 생성 규칙

1. `===PAGE N===` 마커로 구분된 페이지 텍스트에서 직접 인용
2. 15~80자, 문장 중간 절단 금지
3. 가이던스와 직접 관련된 텍스트만
4. 적절한 텍스트 없으면 빈 문자열

### API 호출 파라미터

| 항목 | 값 |
|------|-----|
| 엔드포인트 | `https://api.anthropic.com/v1/messages` |
| 모델 | `claude-sonnet-4-5` |
| 최대 토큰 | 16,000 |
| 문서 최대 길이 | 75,000자 |

### 응답 파싱

```json
{
  "results": [
    {
      "id": "G-A.1__1.1",
      "status": "met | partial | not",
      "score": 0~95,
      "page": 5,
      "excerpt": "보고서 원문 인용",
      "evidence": "판단 근거",
      "rec": "개선 권고사항",
      "notReason": "미이행 사유"
    }
  ]
}
```

## 7. 결과 검증 (환각 제거)

Claude가 생성한 발췌문이 실제 PDF 텍스트에 존재하는지 검증합니다.

### 검증 프로세스

```
Claude 응답 수신
      │
      ▼
발췌문 정규화 (공백·구두점 제거, 소문자 변환)
      │
      ▼
발췌문 앞 30자를 기준으로 탐색
      │
      ├── 명시된 페이지 ±2 범위 탐색 → 발견 시 페이지 번호 교정
      │
      └── 전체 문서 탐색 → 발견 시 실제 페이지로 교정
                │
                └── 미발견 시 → 발췌문 삭제, evidence에 경고 추가
```

### 결과 로깅 예시

```
배치 1: 33개 수신 · ⚠️ 2개 환각 제거
배치 2: 24개 수신 · ⚠️ 0개 환각 제거
```

### 상단 지표 카드 (4개)

| 카드 | 내용 |

### PDF 탭 — 3단 패널

```
┌──────────────────┬──────────────────────┬─────────────────────┐
│   왼쪽 패널      │    가운데 패널        │    오른쪽 패널       │
│   (240px)        │    (나머지 공간)      │    (280px)          │
│                  │                      │                     │
│ [필터 칩]        │ [PDF 툴바]           │ 선택된 가이던스:     │

│                  │                      │ - 상태 배지 + 점수   │
│ [TNFD 트리]      │ [PDF 스크롤 뷰어]    │ - 페이지 번호       │
│ ▸ G-A (6/21)     │  페이지별 캔버스     │ - 발췌문 (노란 배경) │
```

1. 오른쪽 패널에 상세 정보 표시

3. 발췌문 텍스트에 노란색 하이라이트 표시
4. 텍스트 미발견 시 "원문 확인 필요" 메시지 표시

## 10. 데이터베이스 스키마

### `projects` 테이블

```sql
id            UUID PK         -- 자동 생성
user_id       UUID FK         -- auth.users 참조
client_name   text            -- 진단 대상 회사명
bm_client_name text           -- BM 회사명
industry      text            -- 산업
report_year   int             -- 보고서 연도
pdf_filename  text            -- 파일명
bm_filename   text            -- BM 파일명
pdf_path      text            -- Storage 경로 ({user_id}/{job_id}.pdf)
bm_path       text            -- BM Storage 경로
status        text            -- draft | analyzing | done
results       jsonb           -- 진단 결과 JSON
bm_results    jsonb           -- BM 진단 결과 JSON
pdf_texts     jsonb           -- 페이지별 텍스트 캐시
bm_texts      jsonb           -- BM 페이지별 텍스트 캐시
created_at    timestamptz
updated_at    timestamptz     -- 트리거로 자동 갱신
```

### RLS (행 수준 보안)

- SELECT / INSERT / UPDATE / DELETE 모두 `user_id = auth.uid()` 조건
- 본인 데이터만 접근 가능

### Storage 버킷

- 버킷명: `pdfs` (비공개)
- 경로 구조: `pdfs/{user_id}/{job_id}.pdf`
- RLS: 폴더명(user_id)으로 접근 제어

## 12. 보안 정책

| 항목 | 방식 |
|------|------|
| API 키 | DB 저장 안 함, RAM에서만 사용, 잡 완료 즉시 삭제 |
| 비밀번호 | Supabase가 bcrypt 해시 저장 |
| 데이터 격리 | RLS로 본인 데이터만 접근 |
| PDF 격리 | Storage 경로에 user_id 포함, RLS 적용 |
| 브라우저 CORS | `anthropic-dangerous-direct-browser-access: true` 헤더 사용 |
| 비밀번호 복구 | 데모 단계 미구현 (Supabase 대시보드에서 수동 리셋) |
